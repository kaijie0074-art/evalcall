## 六步契约

| 步骤 | 输入 | 执行 | 输出与落盘产物 |
|---|---|---|---|
| 01 上传评测材料 | SOP / System Prompt、整批对话 | 格式校验、标准化、PII 遮罩 | 标准任务包、接入报告、标准 JSONL |
| 02 生成评分标准 | 任务包、全局安全红线 | 编译并合并 L0/L1 规则 | 带等级、类型和原文来源的 checklist |
| 03 检查每通电话 | 整批对话、checklist、裁判票数 | 调用 `python -m evalcall evaluate` | pass / fail / NA、原始证据、异常和复核标记 |
| 04 查看评测结果 | 全部逐项判定 | 统计门禁、失败比例、覆盖与盲区 | 上线决策、问题排名、完整报告 |
| 05 定位问题原因 | 决策结果、SOP lint、裁判与覆盖信号 | 确定性多根因分析 | 原因、置信度、证据、负责人和验证动作 |
| 06 生成修复方案 | 根因、失败证据、候选动作 | 组装修复草案与同尺回归请求 | 修复计划、版本、回跳步骤、checklist hash、回归路径 |

## 真实执行边界

- “已验证结果”读取仓库内真实历史 run 和报告，页面明确标注“历史产物 · 非现场调用”。
- “本地实时运行”的六个步骤都有服务端接口；第 3 步对整批对话执行真实 `evaluate`，不再截取首通，第 4–6 步消费其真实产物。
- 第 6 步只生成草案和回归请求，不自动修改生产 SOP、黄金真值或外呼模型。

## 本轮发现并修复

- 旧页面只有第 3 步会真实评测，其他步骤是前端计时器；已替换为六个后端契约和可下载产物。
