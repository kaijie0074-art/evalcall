## 一、数据攻防卡：我们的数据什么成色

### 什么是真的（三块硬证据）

1. **官方脱敏指令是真的**：两份官方外呼任务指令（xlsx 脱敏数据）零改造接入。**最硬的战绩**：指令体检器只靠官方数据就抓出任务二物理冲突（要求回复≤20字，官方参考话术自身 43 字）——这条发现不依赖任何合成数据，独立成立。
2. **评测流水线的运行是真的**：2026-07-02 用门禁版流水线对官方两任务 + 无冲突对照任务全量真实重跑（真实 LLM 多轮对话、三票判定），产物在 `runs/*_gated_20260702/`，门禁结论/履约率/P0 计数都从真实 run 聚合。
3. **裁判校准是真的**：32 例黄金集 + 22 例盲标注 held-out（标注者与调优链隔离）+ 预注册对账（改崩的迭代被拦截，全文在仓库）。裸裁判对照实验同尺同模型，差距可测量。

### 什么是合成的（主动申报，不等人问）

- **对话的"用户侧"是 LLM 扮演的 persona**（6 种性格），不是真实用户录音。
- **persona 配比是行业常识默认**（平淡为主、极端少数），不是真实话务统计——配比文件里就写着"非真实话务统计"，拿到真实分布可整文件替换。
- 校准黄金集对话为人工构造（构造时埋真值，这是校准方法学的标准做法）。
- `runs/_demo`、`runs/demo-fake` 为模板渲染验证用合成样例，目录内 README 已标注，不作为评测证据。

### 为什么结论可迁移（被问"你评的又不是生产模型"）

- 系统评的是**"模型 × 指令"的遵循关系**，全部判定锚定指令原文（可溯源检查点），不绑定任何被测模型；换被测模型 = 换一个环境变量（TARGET_BACKEND/TARGET_MODEL）。
- 已证明系统**测得出模型差异**：约束消融实验（看得见约束 100% vs 看不见 28.6%）；Haiku vs Sonnet 真实对比（反直觉：强字数约束下 Haiku 更守指令）——说明它不是"对谁都打一样的分"。
- 评测机制（溯源/对抗/校准/门禁）与数据源正交：真实通话进来走同一条判定链。

### 真实数据来了第一天怎么接（D0 路径）

1. 真实通话转写 → 按字段映射表（`data/README.md`）转 `transcripts.jsonl` → `evalcall report` 直接离线评测，**不用重跑对话**。
2. 真实话务分布 → 替换 `data/personas/mix.yaml` 一个文件。
3. 真实坏案例（被投诉通话）→ 加入校准集，重算裁判查准/查全。
4. 业务履约口径 → 改任务 YAML 的 goal 一行。
向导师/评委的诉求：**哪怕 3 条脱敏真实通话**，就能把路径 1 从"理论可行"变成"已演示"。

## 二、数字口径卡：每个数字只有一个出处

|---|---|---|

| **69** | 自动化测试数 | `python3 -m pytest tests/ -q`（2026-07-02 起，含端到端冒烟） |

**"跟裸 GPT 打分啥区别"的答法更新（2026-07-02 对照实验后，按真数据说话）**：
- **别再讲"我们裁判更准"**——同日同模型对照：裸判黄金集 97.0%、对抗集 95.7%，都不低于完整裁判（同日 91.5%）。这是今天强模型的真实水平，如实承认。
- 改讲三层：①**可知性**——"裸判 97%"这个数是我们的校准体系测出来的，没有黄金集+盲测集你连裁判什么时候坏了都不知道（当天实测抓到一次通道故障把数字打到 77.6%，校准一眼定位错误模式）；②**可审计性**——裸判只给结论，我们每条带对话原文证据+指令溯源+置信分流，质检复核与争议仲裁靠证据链；③**系统能力**——门禁/履约/审指令/覆盖率，裸打分全都没有。
- 附加价值（历史数据）：6 月实验证明判定协议能把弱模型抬起来（同模型 3 票 86.5% → 跨模型团 94.2%）——规模化用便宜模型时协议价值放大。

**三条口径纪律**（防自己讲乱）：
1. 说 92.1% 必须带"32 例黄金集·6月跨模型团"；说 94.7% 必须带"held-out 盲测"；说 91.5% 必须带"2026-07-02 同日单模型复测"——三个数不混用。
2. 76.7%/91.7% 是"违规类"的查准/查全，不是总准确率——别和 92.1% 打架。
3. 所有"率"都有产物文件可指——被质疑当场打开 `runs/` 对应 json。
