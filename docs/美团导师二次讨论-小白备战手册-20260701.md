## 第一部分:你的项目到底是个啥(用大白话讲透)

### 1.1 一句话 + 一个比喻

**一句话**:EvalCall 是给"会打电话的 AI 客服"做的**自动监考 + 质检系统**。

**一个比喻**:想象美团有一大批 AI 客服,每天给用户打几百万个电话(催单、改约、安抚)。
- 你的系统就像一个**自动监考老师**:它拿着一份"任务说明书"(规定这通电话该怎么打),派出一群**演员扮演的刁钻顾客**去跟 AI 客服通话,全程录音,然后**逐条打分**:说明书要求的做到没?有没有说脏话/泄露隐私这种红线?这通电话最后到底有没有把事办成?
- 最后出一份**带证据的体检报告**——不是笼统地说"打了 80 分",而是"第 3 条没做到,证据是通话第 5 轮 AI 客服说了这句话"。

**为什么这事值钱**:美团现在靠人工抽听质检——贵(几百万通听不过来)、慢(问题发现滞后)、难量化(每个质检员标准不一样)。你把它**自动化 + 标准化 + 可审计**了。

### 1.2 它解决的三个真问题(你要能脱口而出)

| 痛点 | 大白话 |
|---|---|
| **贵** | 人工抽听,规模一大成本爆炸,全量覆盖几乎不可能 |
| **慢** | 等人听完发现问题,模型都迭代好几版了 |
| **难量化** | 张三标"还行"李四标"不行",没有统一尺子,结果没法追溯 |

## 第二部分:系统五大件(每件都讲清"干啥的 + 凭什么聪明")

你的系统像一条流水线,五个环节。每个我先说**干啥的**,再说**凭什么是亮点**。

### ① 指令编译器(compiler.py)——把"说明书"拆成"勾选清单"
- **干啥的**:任务说明书是一大段自然语言(比如"开场先播报录音提示,再表明身份,严禁承诺具体赔偿金额…")。编译器用大模型把它拆成一条条可勾选的**检查点**,比如:检查点①"是否播报录音提示"、检查点②"是否表明身份"、检查点③"有没有承诺具体赔偿金额(禁止)"。
- **凭什么聪明**:每条检查点都标注**"我是从说明书哪句话来的"(可溯源 source_quote)**。这是你整个项目最核心的卖点——打分有据可查,不是 AI 拍脑袋。导师如果只记一个词,就让他记**"可溯源"**。

### ② 用户模拟器(simulator.py)——派"演员"去刁难 AI 客服
- **干啥的**:AI 扮演各种顾客去跟被测的 AI 客服通话。有 **6 种性格**(术语叫 persona):配合型、不耐烦型、跑题型、隐私警惕型、愤怒辱骂型、分心沉默型。
- **凭什么聪明**:两点。
  - **对抗模式**:演员不是随便聊,而是**专门往红线上诱导**——比如故意逼问"那你赔我多少钱",看 AI 客服会不会上当承诺具体金额(违规)。
  - **覆盖驱动**(coverage-guided):哪条检查点还没被测到,模拟器就**专门制造场景去触发它**,不留死角。

### ③ 对话竞技场(arena.py)——让两个 AI 真的通话
- **干啥的**:让"被测 AI 客服"和"模拟用户"真的来回多轮对话,把完整通话记录录下来。
- **小白点**:这一步没什么花活,就是个"对话执行器",把前面的检查点、演员、被测模型串起来跑。

### ④ 双轨评测引擎(judge.py)——两套裁判一起判
- **干啥的**:读通话记录,逐条判每个检查点"过 / 不过 / 不适用"。
- **凭什么聪明**:这是你第二个核心卖点,**两套裁判**:
  - **规则轨**:机械查关键词(比如有没有出现禁语)。但它只当"探测器"提供线索,不单独定罪——因为"不太好的话"里也含"好的",硬匹配会误杀。
  - **LLM 轨**:让大模型逐条读通话记录判定,**必须引用通话第几轮的原话当证据**。而且是 **3 票多数投票**,甚至能让**不同的大模型各投一票**(haiku、sonnet、opus)——这叫**跨模型裁判团**,目的是消除单个裁判的系统性偏见(同一个模型投 3 次只能消随机噪声,换不同模型投才能消偏见)。

### ⑤ 评测报告(report.py)——出一份带证据的体检报告
- **干啥的**:把判定结果聚合成一份 HTML 报告:四维雷达图、逐条明细(结论+证据+置信度)、失败案例完整回放(违规轮次高亮)、按用户类型切片。
- **新增(这次加的)**:报告头条直接给**上线决策"可上线/打回"**、**履约达成率**、**待人工复核数**,还有**通话级清单**(一通一通看)。

## 第三部分:项目的灵魂——"三层质量闭环"(这是你区别于"又一个打分器"的根本)

导师一定会想:"这不就是拿 GPT 给对话打个分吗?" 你的回答是**三层闭环**,回答"凭什么信你的评测":

| 层 | 回答的问题 | 怎么做的 |
|---|---|---|
| **评模型** | AI 客服守不守说明书? | 上面那条流水线 |
| **审指令** | 分低,是模型差,还是**说明书本身有病**? | **指令体检器**(lint.py)自动查说明书里的自相矛盾/不可行/歧义。**你的杀手锏战绩**:自主发现官方任务二的物理冲突——说明书一边要求"回复≤20字",一边给的参考话术是43字,**物理上不可能同时满足**,所以模型必然违规、必然 0 分。这不是模型差,是题目出错了。 |
| **校裁判** | 你的裁判自己判得准吗? | **校准器**(calibrate.py):用一批"埋好正确答案"的对话去考裁判,算出**查准率/查全率/混淆矩阵**。你的报告自带"误差棒",不装得自己 100% 对。 |

> **这三层是你最该让导师记住的东西**。一句话总结:**"做个评测器不难,难的是凭什么信它——我用'审指令'和'校裁判'两层,把'评测器本身可不可信'也量化了。"**

## 第四部分:这次按导师建议做了啥(开场就亮,显示你听进去了)

上次导师给了建议,你这次**真的落地了 16 项改进**(59 个自动化测试全过)。开场就说这个,气场完全不一样。挑最能打的讲:

1. **上线红线门禁**:报告不只给分数,直接给**"可上线 / 打回"的决策**——任一 P0(安全底线)问题就打回。(对应导师说的 P0/P1/P2 分级)
2. **安全/合规红线轨**:辱骂、歧视、隐私泄露、未告知 AI 身份、诱导——这些安全底线独立成**最高优先级一票否决轨**。(对应导师说的"P0=直接阻断")
3. **履约达成检查点**:新增测"**这通电话到底有没有把活办成**"(改约成功没/安抚到位没)——这是真实外呼第一 KPI。(这条是质询时"业务视角"挖出来的,导师会很有共鸣)
4. **真实性检查点**:测"被用户质疑'你是不是机器人'时,AI 客服会不会崩"。(对应导师说的"是否被识别为 AI")
5. **跨版本回归对比**:模型改了一版,自动告诉你"这版比上版好了还是坏了"。
6. **活清单**:能从真实对话里自动发现"说明书要求了但检查清单漏掉的"检查点(且有防作弊的硬闸)。

## 第五部分:小白术语词典(导师提到这些词,你不会卡壳)

| 术语 | 大白话 |
|---|---|
| **检查点(checkpoint)** | 把说明书拆成的一条条可勾选的小要求 |
| **可溯源(source_quote)** | 每条检查点都标"来自说明书哪句话",打分有据可查 |
| **persona** | 模拟用户的"性格人设",你有 6 种 |
| **对抗模拟** | 演员专门往红线上诱导,测 AI 会不会上当 |
| **覆盖率(coverage)** | 检查点被测到的比例;盲区=从没被任何对话测到的检查点 |
| **双轨判定** | 规则(机械查词)+ LLM(大模型读+引证据)两套裁判 |
| **LLM-as-judge** | 用大模型当裁判判对话 |
| **多数投票 / 裁判团** | 一条检查点判 3 次取多数;甚至不同模型各投一票消偏见 |
| **一票否决(critical fail)** | 出现严重违规,该项直接 0 分 |
| **severity(严重度)** | 检查点分 critical/major/minor,对应业务 P0/P1/P2 |
| **查准率(precision)** | 裁判判"违规"的里头,真的违规占多少(防冤枉) |
| **查全率(recall)** | 真的违规的里头,裁判抓到多少(防漏判)。**安全项看的是查全率**——宁可错杀不可放过 |
| **校准(calibrate)** | 用埋好答案的题考裁判,算它准不准 |
| **held-out 盲测集** | 一批"标注时裁判没见过"的题,防止裁判是"背答案"而非"真会判" |
| **预注册** | 改进前先写死"我预计提升多少",跑完对账,防止自欺 |

**你的关键数字(背下来)**:裁判准确率 **92.1%**(32 题全量)/ **94.7%**(held-out 盲测)、违规查全率 **100%**、覆盖率 **100%**、评委同尺四轮 30→32。

## 第六部分:跟导师聊什么(这是本手册最值钱的部分)

核心策略:**问那些"他答得上、你答不上、Google 也搜不到"的问题**——也就是他的真实业务 know-how。别问技术细节(那是你的强项),要问**真实外呼业务长什么样**。

### A. 主动问导师的高价值问题(榨取他的业务经验)

**【关于真实业务,校准你的系统】**
1. "真实外呼里,**什么样的问题最致命、最常被用户投诉或被监管盯**?——我想用它校准我的 P0 红线和严重度权重,别把劲使错地方。"
2. "美团内部现在**外呼质检是怎么做的**?人工抽听大概抽多少比例?**最想自动化掉哪一块**?——我想让我的系统对得上真实痛点,而不是自嗨。"
3. "**'履约达成'在真实业务里具体怎么算'办成了'**?比如一通改约电话,是用户口头同意算成,还是系统里订单真改了才算?——这直接决定我的'履约检查点'判据怎么写。"

**【关于数据】**
4. "脱敏数据**大概什么时候能给、长什么样**?有没有真实的'坏案例'(被投诉的通话)?——哪怕几条,对我校准裁判都极有价值。"
5. "真实外呼用户里,**情绪差/辱骂的占比大概多少**?——我现在模拟器默认按'平淡为主'配比,但拍脑袋的,想按真实分布校准。"

**【关于评估维度,查漏】**
6. "美团评估一个对话模型好不好,**除了'守不守指令',还看哪些维度**?——我怕有我没覆盖到的盲区(比如通话时长?用户满意度?转人工率?)。"

**【杀手锏:C12 合规问题,显示你有独立判断】**
7. "您上次建议测'是否被识别为 AI'。我研究后发现一个**张力**:履约外呼很多合规场景**反而要求主动表明 AI 身份**,藏身份可能踩线。所以我实现成了'被质疑时 AI 能否稳健应对、该表明身份就表明',而不是'维持人设骗过用户'。**您怎么看真实业务里这块的合规边界**?到底是该藏还是该亮?"
   - 这条问题是核武器:它同时证明①你消化了他的建议 ②你有自己的判断 ③它能引出一大段他的业务/合规经验。

### B. 导师可能反问你的(提前备好答案,别被问住)

| 他可能问 | 你这样答 |
|---|---|
| "你这跟直接用 GPT 打个分有啥区别?" | "三点:①每条结论可溯源到说明书原文,可审计;②不是随机聊天,是对抗式诱导找漏洞;③裁判自己经过校准、还跨模型投票消偏见。" |
| "你怎么保证裁判判得准?" | "用埋好答案的黄金集考它:查准 76.7%、查全 91.7%,held-out 盲测 94.7%。报告自带误差棒,不装 100% 对。" |
| "官方任务为什么全 0 分?是你系统不行?" | "**恰恰相反——是我系统抓到了题目本身的物理冲突**:说明书要求≤20字但参考话术43字,不可能同时满足。这是'审指令'层的功劳,0 分归因于指令缺陷,不是模型差也不是评测失灵。" |
| "你的对抗模拟会不会造现实里不存在的极端场景?" | "会注意这点——模拟器配比可配置,默认按行业常识'平淡为主',不把辱骂型放大,避免模型过拟合罕见场景。配比口径我都如实标注了。" |
| "一票否决会不会太严,好模型也被判死?" | "分级了:只有 P0(安全红线)才一票否决,P1/P2 是扣分和复核。" |

### C. 怎么把对话引向深入(技巧)

1. **开场亮进展,不要示弱**:不说"我不太懂请您指点",说"上次您的建议我落地了 16 项,今天想跟您深聊几个我拿不准的业务问题"。姿态从"学生"变"合作者"。
2. **问开放题,逼出经验**:每个问题后追一句"**能举个真实例子吗**"——细节里才有金子。
3. **主动暴露不确定**(这叫弱点先发权):"我这块是拍脑袋定的,您帮我校准下"——比假装都懂更能引出他的干货,也显得你诚实。
4. **带着你的'答不上清单'去**:把上面 B 区你答得最虚的那条,直接抛给他请教。
5. **留一个钩子**:聊到一半抛 C12 那个合规问题,基本能撑起后半场深聊。
