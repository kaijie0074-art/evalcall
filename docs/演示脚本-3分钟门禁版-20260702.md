质检员的工作流就是这张通话级清单：哪通踩了红线、点进去看原文证据——**不用重听录音**。

**[切主页指令体检区块]**

**[口述]**

实测时官方任务二全员归零。一般评测系统到这就结束了：「模型不行」。

我们的指令体检器发现：**根因不在模型，在指令**——指令要求「每次回复最多 20 字」，官方参考话术自己就有 43 字。物理冲突，任何模型都做不到。报告 0 分旁边自动挂归因徽章：**这是题目的病，不是模型的病，更不是评测器坏了**。

对照实验补一刀：同一套流水线跑**无冲突的改约任务**，分数正常分布、有区分度——证明 0 分是指令缺陷的精确检出，不是系统只会给 0 分。

对美团来说这是上线前的指令体检环节：**先治指令，再评模型，质检数据才有意义。**

## 02:00–02:45 | 第三层 · 校裁判 + 裸裁判对照（45秒）

**[切校裁判区块]**

**[口述]**

有人会问："这跟直接拿 GPT 打个分有什么区别？"我们真做了正面对照实验，**结论先如实说：在今天的旗舰模型上，裸 LLM 单票判黄金集准确率 97.0%，很高**——但这恰恰是我们最硬的一张牌，三层递进：

第一，**这个 97% 是谁测出来的？是我们的校准体系**——32 例埋好真值的黄金集加 22 例盲标注对抗集。没有这套基础设施，你连"裸判到底准不准、什么时候会坏"都无从知道。今天上午我们的校准还实测抓到一次裁判通道故障（数字掉到 77.6%），错误模式一眼定位——**校准是评测系统的照妖镜，照别人也照自己**。

第二，**准确率相当≠产品能力相当**：裸判只给"过/不过"，我们每条结论带对话原文证据、溯源到指令原句、置信度分流待人工复核——质检员复核、跟业务方争议仲裁，靠的是证据链不是准确率。

第三，**评测系统远不止裁判**：上线门禁、履约达成、指令体检（抓出官方指令物理冲突）、覆盖率驱动——这些裸打分一样都没有。

## 备用应对（新增三条 + 保留旧版全部）

**Q：你评的是 Claude，不是外呼生产模型，结论凭什么迁移？（数据迁移性，必被问）**
A：三步答：①系统评的是"模型×指令"的**遵循关系**，不是模型本身——检查点、判定、门禁全部锚定指令原文，换被测模型零改造（TARGET_BACKEND 一个环境变量）；②我们已用受控实验证明它能区分不同模型（约束消融 100% vs 28.6%；Haiku vs Sonnet 反直觉结论）；③真实数据来了第一天就能接：离线轨迹格式适配层已就位，官方脱敏 xlsx 零改造接入就是实证。**同时如实说：合成对话的用户分布是行业常识配比，非真实话务统计——这是我们主动标注的已知局限，拿到真实分布即可替换配比文件。**

**Q：履约达成"办成了"的判据谁定的？**
A：来自任务 YAML 的 goal 字段（从指令原文提炼、可溯源），判定引用对话原文证据。但"口头同意算成还是系统改单才算成"是业务口径问题——这正是我们向业务导师请教后待对齐的点，判据文本可按真实业务口径一行替换。（诚实暴露+引导师背书）

**Q：跟开源评测框架（τ-bench 之类）的区别？**
A：τ-bench 评的是"任务完成"（工具调用轨迹对不对），我们评的是**对话过程合规 + 任务完成双轨**，且多三层：指令体检（题目本身有没有病）、裁判校准（判得准不准自带误差棒）、上线门禁（结论落到能不能上线）。学术对标详见主页对标区块。
