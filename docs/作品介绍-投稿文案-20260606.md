# EvalCall — 外呼对话模型指令遵循自动评测系统

## 项目背景与痛点

美团履约数字人每天产生大规模外呼对话。任务指令往往包含复杂的流程分支和多重合规约束——关键信息播报顺序、禁止话术、隐私保护要求、情绪处理规范等。当前质检高度依赖人工抽听：全量覆盖成本不可承受，问题发现滞后无法支撑快速迭代，评分标准因人而异难以追溯。

EvalCall 解决的核心问题：**如何把"人工看对话有没有按指令来"这件事，变成可量化、可重复、可审计的自动化流程。**

---

## 创新性

EvalCall 的核心创新在于四个相互咬合的设计：

**一、指令→检查点编译，评测有根可溯。** 系统首先把自然语言任务指令编译成结构化检查点清单，每条检查点标注类型（流程节点/硬约束/禁止项/话术要求）和 `source_quote`（可溯源到指令原文）。评测结论不是"大模型觉得还行"，而是逐条对应指令原文的通过/失败，有原文依据，可审计、可复现。

**二、对抗式模拟器 + 覆盖率驱动，测试充分有效。** 用户模拟器不是随机聊天，而是主动针对检查点中的约束项"定向诱导"被测模型违规，同时追踪检查点覆盖率，优先制造尚未被触发的场景。六种 persona（配合型/打断型/跑题型/质疑型/情绪型/沉默型）全面覆盖真实用户行为。

**三、双轨判定 + 多数投票 + 自一致率，结果可靠可信。** 规则轨处理确定性判断（关键词是否出现、禁语是否触发），LLM 轨处理语义复杂场景（须引用对话原文作为证据），3票多数投票降低单次误判风险。系统同时输出 judge 自一致率和规则/LLM 双轨冲突率，让评测结果自带可靠性度量。

**四、数据即插即用，零侵入接入。** 格式适配层将外部脱敏数据映射到内部 schema，评测逻辑与数据格式彻底解耦。

---

## 完整性

系统完整交付赛题要求的两个核心目标：

- **用户模拟器**：多 persona × 多策略 × 对抗模式，能充分有效测试对话模型在特定任务指令下的表现。
- **评测报告**：逐检查点明细（结论 + 证据引用 + 置信度）、四维雷达图（流程完整度/约束遵循率/异常处理/话术合规）、失败案例剖析、persona 维度切片，过程可解释、结果可量化。

端到端流程完整闭环：任务 YAML → 指令编译 → 多轨迹模拟 → 批量判定 → HTML/JSON 报告，两条命令（`evalcall run` / `evalcall report`）覆盖全流程。内置五类模拟场景（外卖催单确认、配送时间预约、商家回访、超时安抚、隐私合规），无需真实数据即可完整演示。

## 应用效果

| 指标 | 说明 |
|------|------|
| 约束违反数/百次对话 | 系统核心量化产出，可按 task / persona / severity 三维切片，替代人工"大概违反了几次"的模糊印象 |
| 评测成本对比人工 | 单条轨迹全量判定约数秒，批量并发下成本较人工抽听降低数量级，且覆盖率100% |
| 可解释证据覆盖率 | 每条 fail 结论必须附对话原文引用，证据覆盖率 = 有证据的 fail 数 / 总 fail 数，目标 > 95% |

评测报告中的"失败案例剖析"直接给出：哪条对话、哪一轮、哪个检查点失败、原文证据是什么。质检员无需重听录音，一个报告页完成全量复核。

## 商业价值

美团履约外呼是高频、高合规要求的业务场景。EvalCall 的商业价值体现在三个层次：

**一、质检自动化降本。** 外呼规模越大，人工质检的成本优势越显著。系统支持全量覆盖，消除抽样盲区，问题可在上线前批量发现，而非等投诉驱动。

**二、模型迭代加速。** 每次模型版本更新后，一次评测跑出量化对比报告，直接看约束遵循率的变化趋势，研发团队无需等待人工评估窗口。

**三、横向复用。** 框架不绑定外卖催单场景，客服外呼、营销外呼、预约提醒等只需更换任务 YAML，评测逻辑全部复用。指令编译器和双轨判定引擎是通用组件，可接入任何基于任务指令的对话系统质检场景。
