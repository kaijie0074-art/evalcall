# EvalCall 决赛提交说明

## 项目定位

自动测试并评估外呼模型在特定任务指令下的指令遵循效果。外呼模型是被评对象；SOP 是评分依据；用户模拟器是测试工具；对话日志是过程证据；SOP 诊断、裁判校准和根因归因用于保障结论可信。

## 现场演示建议

优先使用本地实时模式展示六步完整链路；实时模型超时或现场网络不稳定时，明确切换到“已验证缓存”，继续展示相同版本和 hash 的完整产物。缓存结果不能口述成现场新生成结果。

主案例使用“配送时间改约”，在固定 SOP / Checklist 下展示外呼模型首要根因；对照案例使用“低延迟直播升级”，展示系统能识别 SOP / 指令问题，证明不会把所有失败都归给模型。

## 最终数据口径

- 主案例 v1：10 通、21 项、210 条判断、3/10 打回、履约率 40%、P0 3/10、平均分 62.5，首要根因 `target_model`。
- 同尺回归 v2：同一用户输入、SOP、Checklist 与 Judge；门禁“可上线”、P0 0/10、履约率 40%、平均分 91.4；正常重评且未手改 judgments。
- 对照案例：10 通、9 通打回、SOP 可行性 0/100，首要根因 `instruction`。
- 裁判校准：32 case、165 条真值、总体准确率 92.73%、P0 fail recall 100%。

以上数字均来自仓库中的正式 run 与校准产物，不把缓存结果描述为实时结果，也不把单通结果外推为多通结果。
